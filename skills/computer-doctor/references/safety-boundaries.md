# Safety Boundaries

## Audit mode

- default to read-first behavior
- report findings before changing things
- ask before making destructive or workflow-changing actions

## YOLO mode

- proceed without repeated approval after the initial mode choice
- still avoid reckless actions
- do not touch personal files, project folders, credentials, browser profiles, databases, cloud-sync folders, or external drives unless the action is clearly relevant and low-risk

## Shared safety rules

- prefer evidence over guesses
- call out uncertainty when confidence is weak
- if an action could remove data or create hard-to-reverse damage, treat that as high risk even in YOLO mode
