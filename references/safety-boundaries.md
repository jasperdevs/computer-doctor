# Safety Boundaries

This skill is audit-first and non-destructive by default.

- Do not make destructive changes without explicit approval.
- Do not touch personal files, project folders, credentials, browser profiles, databases, cloud-sync folders, or external drives unless specifically approved.
- If a change could remove data, break workflows, or is hard to undo, stop and ask first.
- Prefer reporting and recommending over fixing automatically.
- If evidence is partial, say so explicitly.
